## 1. 플랫폼 개요

Pochak은 3개의 사용자 영역으로 구성됩니다.

### 1.1 포착 TV

| 항목 | 정책 |
|------|------|
| 목적 | 개인화 페이지 및 관리자 전시 영역 (유저 타겟) |
| 대상 | 모든 사용자 (비로그인 포함) |
| 콘텐츠 | 홈 피드, 추천 콘텐츠, 인기 영상, 검색 결과 |
| 구성 | 관리자가 BO에서 배너/섹션/추천 알고리즘을 통해 전시 관리 |
| 개인화 | 로그인 사용자에게 시청 이력/선호 종목 기반 추천 제공 |

### 1.2 포착 시티

| 항목 | 정책 |
|------|------|
| 목적 | 체육시설 정보 / 협회 및 지자체 정보 공개 (지역 기반) |
| 대상 | 모든 사용자 (비로그인 포함 열람 가능) |
| 표시 단체 | 공개된 체육시설, 지자체 시설, 대규모 협회 |
| 핵심 기능 | 시설 예약 현황 조회, 촬영 예약, 시설 정보 열람 |
| 인증 요건 | 지자체/대규모 협회의 경우 **BO에서 관리자 인증 필수** (신뢰성 확보) |
| 콘텐츠 공개 | 기본 PUBLIC (누구나 열람 가능) |

### 1.3 포착 클럽

| 항목 | 정책 |
|------|------|
| 목적 | B2B / B2C 영업 대상 단체 표기 장소 |
| 대상 | 시티/TV를 제외한 **모든 단체** |
| 콘텐츠 | CUG(Closed User Group) 단체: 콘텐츠 직접 노출 불가, 단체 자체는 접근 가능 |
| CUG 정책 | 회원 전용 영상과 홍보용(비회원) 영상을 **별도 관리** 가능 (YouTube 멤버십 모델) |
| 커뮤니티 | 소식 / 구인 / 모집 등 커뮤니티 성 공간 제공 (지역 기반, 당근마켓 스타일) |

### 1.4 영역 배치 규칙

```
포착 TV ─── 모든 사용자의 메인 진입점, 개인화 피드
  │
  ├── 포착 시티 ─── 공공 체육시설 / 협회 / 지자체 (지역 탐색)
  │     └── 예약 현황, 시설 정보, 공개 콘텐츠
  │
  └── 포착 클럽 ─── B2B/B2C 단체 (가입 기반)
        ├── 오픈 단체: 자유 가입, 콘텐츠 공개
        └── CUG 단체: 단체는 보이되, 콘텐츠는 회원만
              ├── 회원 전용 영상 (가입 후 접근)
              └── 홍보용 영상 (비회원도 열람 가능)
```

## 2. 회원 정책

### 2.1 계정 식별자

| 항목 | 정책 |
|------|------|
| 국내 사용자 식별자 | 전화번호 1개 = 1 계정 (UNIQUE) |
| 해외 사용자 식별자 | 이메일 1개 = 1 계정 (UNIQUE) |
| OAuth 연동 | 1 계정 : N 로그인 방식 (카카오, 구글, 네이버 동시 연결 가능) |
| loginId | 6~20자, 영문+숫자+밑줄(_)만 허용, 중복 불가 |
| 비밀번호 | 8~20자, 3종류 이상 조합 (영문/숫자/특수문자) |
| 이메일 | 제공 시 UNIQUE, 대소문자 구분 없음 |

### 2.2 가입 경로 (4개)

#### Route A: 국내 성인 (14세 이상)

```
이용약관 동의 → 휴대폰 인증 → [중복 확인] → 계정 정보 입력 → 추가 정보 → 가입 완료
```

| 항목 | 정책 |
|------|------|
| 연령 제한 | 만 14세 이상만 가입 가능 |
| 필수 인증 | 휴대폰 SMS 인증 |
| 필수 입력 | loginId, 비밀번호, 이름, 생년월일 |
| 선택 입력 | 이메일, 성별, 프로필 이미지 |
| 동의 항목 | 이용약관(필수), 개인정보(필수), 마케팅(선택) |

#### Route B: 국내 미성년자 (14세 미만)

```
이용약관 동의 → 보호자 인증 → 본인 휴대폰 인증 → 계정 정보 입력 → 추가 정보 → 가입 완료
```

| 항목 | 정책 |
|------|------|
| 연령 제한 | 만 14세 미만 |
| 보호자 조건 | 만 14세 이상의 성인 계정 보유자 |
| 보호자당 미성년 계정 수 | 최대 5개 (개별 조정 가능) |
| 보호자 생성 제한 | 24시간 내 최대 2개 계정 생성 |
| 미성년자 보호자 불가 | 미성년 계정은 보호자가 될 수 없음 |

#### Route C: SNS 소셜 가입

```
OAuth 인증 → 이용약관 동의 → 휴대폰 인증 → [기존 계정 연결/신규 생성] → 추가 정보 → 가입 완료
```

| 항목 | 정책 |
|------|------|
| 지원 SNS | 카카오, 구글, 네이버 |
| 전화번호 인증 | 필수 (국내 사용자) |
| 기존 이메일 매칭 | OAuth 이메일과 동일한 계정 발견 시 "연결하시겠습니까?" 안내 |
| 프로필 자동 반영 | OAuth에서 받은 이름, 프로필 이미지 자동 적용 |

#### Route D: 해외 사용자

```
이용약관 동의(EN) → 이메일 인증 → 계정 생성 (ID/PW/생년월일) → 추가 정보 → 가입 완료
```

| 항목 | 정책 |
|------|------|
| 인증 방식 | 이메일 인증 (SMS 불가) |
| 국적 입력 | 필수 |
| 언어 | 영문 기본 |

### 2.3 전화번호 인증 규칙

| 항목 | 값 |
|------|---|
| 인증번호 자릿수 | 6자리 |
| 인증번호 유효 시간 | 180초 (3분) |
| 코드당 최대 시도 횟수 | 5회 |
| 시간당 최대 발송 횟수 | 5회/전화번호/시간 |
| 인증 성공 시 | verifiedToken (UUID) 발급, 30분 유효 |

### 2.4 사용자 상태 생명주기

```
UNVERIFIED ──→ ACTIVE ──→ DORMANT_PENDING ──→ DORMANT
                 │              │                  │
                 │              └── 로그인 ──→ ACTIVE
                 │
                 ├──→ BLOCKED (관리자 차단)
                 │       └──→ ACTIVE (관리자 해제)
                 │
                 └──→ WITHDRAWN (본인 탈퇴 요청)
```

| 전환 | 조건 |
|------|------|
| ACTIVE → DORMANT_PENDING | 90일 미접속 |
| DORMANT_PENDING → DORMANT | PENDING 후 180일 경과 |
| ACTIVE → WITHDRAWN | 사용자 탈퇴 요청, 7일 쿨다운 기간 |

### 2.5 사용자 관계

| 관계 유형 | 설명 | 검증 방식 |
|-----------|------|----------|
| FAMILY | 부모, 배우자, 형제 | 상호 확인 |
| COACH | 코치/강사 관계 | 상호 확인 |
| GUARDIAN | 법적 보호자 (미성년) | 가입 시 인증 |
| FRIEND | 비공식 팔로우 관계 | 검증 불필요 |

## 4. 단체 정책

> **v2 변경**: 기존의 ASSOCIATION/PUBLIC/PRIVATE 분류를 폐지하고, **단일 Entity "단체"** 로 통합. 자가참조 DAG 구조로 계층을 표현합니다.

### 4.1 단체 데이터 모델

```
organizations (단일 테이블)
├── id: BIGSERIAL PK
├── parent_id: BIGINT (자가참조, nullable) ──→ DAG 구조
├── name: VARCHAR(200) NOT NULL
├── sport_id: INT
├── description: TEXT
├── logo_url: VARCHAR(500)
├── si_gun_gu_code: VARCHAR(10)    ──→ 지역 기반 (포착 시티/클럽 영역 결정)
│
│ ── 영역 설정 ──
├── display_area: ENUM('CITY', 'CLUB')  ──→ 포착 시티 / 포착 클럽 배치
├── is_verified: BOOLEAN DEFAULT false   ──→ 시티 인증 여부 (BO 관리자 인증)
│
│ ── 접근 설정 (관리자가 "단체 설정"에서 변경) ──
├── reservation_policy: ENUM('ALL_MEMBERS', 'MANAGER_ONLY') DEFAULT 'MANAGER_ONLY'
├── can_host_competition: BOOLEAN DEFAULT true
├── join_policy: ENUM('OPEN', 'APPROVAL', 'INVITE_ONLY') DEFAULT 'APPROVAL'
├── default_content_visibility: ENUM('PUBLIC', 'MEMBERS_ONLY') DEFAULT 'MEMBERS_ONLY'
│
│ ── CUG 설정 (포착 클럽) ──
├── is_cug: BOOLEAN DEFAULT false   ──→ CUG 단체 여부
│     (CUG=true: 단체 자체는 검색/접근 가능, 콘텐츠는 회원만)
│     (CUG=false: 설정에 따라 콘텐츠 공개)
│
├── active: BOOLEAN DEFAULT true
├── created_at, updated_at, deleted_at
```

### 4.2 자가참조 DAG 구조

```
A 협회 (parent_id=null)
├── B 시도체육회 (parent_id=A)
│   ├── C 시군구체육회 (parent_id=B)
│   └── D 구장운영팀 (parent_id=B)
├── E 산하팀 (parent_id=A)
└── F 지점 (parent_id=A)
    └── G 하위지점 (parent_id=F)
```

| 항목 | 정책 |
|------|------|
| 구조 | 단일 Entity, 자가참조 parent_id (DAG) |
| 순환참조 | **금지** - 저장 시 순환 감지 로직 필수 |
| 깊이 제한 | 최대 5단계 (성능/UX) |
| 루트 단체 | parent_id = null |
| 독립 운영 | 어떤 단체든 독립(parent_id=null) 가능 |

### 4.3 포착 시티 vs 포착 클럽 배치 규칙

| 속성 | 포착 시티 (display_area=CITY) | 포착 클럽 (display_area=CLUB) |
|------|---------------------------|--------------------------|
| 목적 | 공공 체육시설, 협회, 지자체 | B2B/B2C 영업 단체 |
| 인증 | **is_verified=true 필수** (BO 관리자 인증) | 인증 불필요 |
| 가입 정책 | 기본 OPEN (누구나 가입) | 기본 APPROVAL (승인 필요) |
| 예약 권한 | 기본 ALL_MEMBERS (모든 멤버 예약 가능) | 기본 MANAGER_ONLY (매니저만) |
| 콘텐츠 공개 | 기본 PUBLIC | 기본 MEMBERS_ONLY |
| 커뮤니티 | 시설 정보, 예약 현황 위주 | 소식/구인/모집 커뮤니티 |

> **핵심**: 시티/클럽은 데이터 분류가 아닌 **표시 영역**입니다. 동일한 단체 Entity를 `display_area` 필드로 구분합니다. 관리자(ADMIN 역할)가 "단체 설정" 화면에서 모든 설정을 변경할 수 있습니다.

### 4.4 CUG (Closed User Group) 단체 정책

| 항목 | 정책 |
|------|------|
| 단체 접근 | CUG 여부와 무관하게 **단체 자체는 누구나 접근/검색 가능** |
| 콘텐츠 접근 | CUG=true인 단체의 콘텐츠는 **회원만 접근 가능** |
| 홍보용 영상 | CUG 단체도 **홍보용(PUBLIC) 영상은 별도 지정하여 비회원에게 공개** 가능 |
| 회원 전용 영상 | visibility=MEMBERS_ONLY로 지정된 콘텐츠는 가입 회원만 열람 |
| 비유 | YouTube 채널 멤버십 모델 (무료 공개 영상 + 멤버 전용 영상 공존) |

```
CUG 단체의 콘텐츠 구분:
├── 홍보용 영상 (visibility=PUBLIC)     → 누구나 시청 가능
├── 회원 전용 영상 (visibility=MEMBERS_ONLY) → 가입 회원만 시청
└── 비공개 영상 (visibility=PRIVATE)    → 지정 사용자만 시청
```

### 4.5 단체 설정 (매니저 이상 접근)

> **v2 변경**: 가입 승인, 예약 권한, 대회 개설 권한 등을 **"단체 설정" 화면**에서 관리자(ADMIN 역할)가 직접 설정합니다.

| 설정 항목 | 설명 | 변경 가능 역할 |
|-----------|------|-------------|
| join_policy | 가입 정책: OPEN / APPROVAL / INVITE_ONLY | ADMIN |
| reservation_policy | 예약 권한: ALL_MEMBERS / MANAGER_ONLY | ADMIN |
| can_host_competition | 대회 개설 가능 여부 | ADMIN |
| default_content_visibility | 신규 콘텐츠 기본 공개 범위 | ADMIN |
| is_cug | CUG 모드 활성화 | ADMIN |
| display_area | 포착 시티/클럽 영역 배치 (BO 관리자만) | BO 관리자 |
| is_verified | 시티 인증 여부 (BO 관리자만) | BO 관리자 |

## 5. 멤버십 & 권한 정책

### 5.1 멤버십 역할

| 역할 | 대상 | 권한 범위 |
|------|------|----------|
| ADMIN | 단체 | 모든 관리 권한, 단체 설정 변경, 멤버/매니저 관리 |
| MANAGER | 단체 | 콘텐츠 관리, 멤버 승인/거절, 예약 관리 |
| COACH | 단체 | 팀 콘텐츠 접근, 선수 관리 |
| PLAYER | 단체 | 팀 콘텐츠 접근, 개인 클립 생성 |
| GUARDIAN | 단체 | 보호 대상 미성년 선수의 콘텐츠 접근 |
| MEMBER | 단체 | 기본 콘텐츠 접근 |

### 5.2 가입 승인 플로우

| join_policy | 사용자 행동 | 처리 |
|-------------|-----------|------|
| OPEN | "가입" 클릭 | 즉시 APPROVED (MEMBER 역할 부여) |
| APPROVAL | "가입 신청" 클릭 | PENDING → MANAGER 이상이 승인/거절 |
| INVITE_ONLY | 가입 버튼 없음 | MANAGER 이상이 초대링크/코드 발급 → 사용자 수락 |

> 가입 정책은 **관리자(ADMIN 역할)가 "단체 설정" 화면에서 변경** 가능합니다.

### 5.3 단체 설정 접근 권한

| 메뉴 | 접근 가능 역할 |
|------|-------------|
| 단체 정보 수정 | ADMIN |
| 단체 설정 (가입/예약/대회/CUG) | ADMIN |

## 6. 대회 정책

> **v2 변경**: 기존에는 "협회만 대회 개설 가능"이었으나, 이제 **모든 단체의 관리자(ADMIN)가 대회를 개설할 수 있습니다.**

### 6.1 대회 개설

| 항목 | 정책 |
|------|------|
| 개설 권한 | 단체의 **ADMIN 역할** 사용자 (단체 설정에서 can_host_competition=true인 경우) |
| 개설 가능 단체 | 모든 단체 (기본 can_host_competition=true) |

### 6.2 대회 공개 범위

| 공개 유형 | 설명 |
|-----------|------|
| **전체 공개** | 대회 카드 목록에 표출, 누구나 검색/접근 가능 |
| **비공개 (초대제)** | 대회 카드 목록에 표출되지 않음. **링크(URL) 또는 QR코드로 초대된 사용자만 접근 가능** |

### 6.3 비공개 대회 접근 규칙

| 항목 | 정책 |
|------|------|
| 접근 방법 | URL 링크 또는 QR코드를 통해서만 진입 |
| 최초 접근 | 비공개 대회 링크를 클릭하면 해당 대회 페이지 진입 |
| 재접근 | **한 번이라도 접근한 사용자는 "방문한 페이지"에서 해당 대회를 확인 가능하며, 재입장 가능** |
| 방문 기록 | `competition_visits` 테이블에 (user_id, competition_id, first_visited_at) 기록 |
| 검색 | 비공개 대회는 일반 검색 결과에 노출되지 않음 |
| 콘텐츠 | 대회 내 경기 영상도 동일한 비공개 정책 적용 |

### 6.4 대회 상태

```

               ↓
           CANCELLED
```

## 8. 콘텐츠 접근 제어 정책

### 8.1 2단계 권한 모델

```
사용자 → [1단계 RBAC: 구독/패스 보유?] → [2단계 ABAC: ACL 정책 충족?] → 재생 허용/거부
```

### 8.2 1단계: RBAC (시청 권한)

| 유형 | 범위 | 우선순위 |
|------|------|---------|
| SUBSCRIPTION | 전체 콘텐츠 (기간 내) | 1 (최우선) |
| SEASON_PASS | 특정 대회 시즌 | 2 |
| MATCH_TICKET | 특정 경기 1건 | 3 |
| POINT_VIEW | 포인트 차감 시청 | 4 |

### 8.3 2단계: ABAC (영상별 접근 제어)

```
1. blockedUsers에 포함? → 즉시 DENY
2. allowedUsers에 포함? → 즉시 GRANT
3. default_policy 평가:
   ├── PUBLIC → ALLOW
   ├── AUTHENTICATED → 로그인 확인
   ├── SUBSCRIBERS → Commerce Entitlement 확인
   ├── MEMBERS_ONLY → 멤버십 확인 (CUG 단체 포함)
   └── PRIVATE → allowedRoles/Relations/Orgs 확인
4. user_relations 확인 (가족, 코치, 보호자)
5. valid_from / valid_until 시간 범위 확인
6. 최종 결정 → 재생 토큰 발급 또는 DENY
```

### 8.4 CUG 단체 콘텐츠 접근 흐름

```
비회원이 CUG 단체 페이지 접근:
  → 단체 정보(이름, 설명, 종목, 멤버 수) 표시 ✓
  → 홍보용 영상 (visibility=PUBLIC) 표시 ✓
  → 회원 전용 영상 → "회원 가입 후 시청 가능합니다" 안내 + 가입 유도 ✗
  → 비공개 영상 → 목록에 표시되지 않음 ✗
```

| 항목 | v1 (기존) | v2 (변경) |
|------|----------|----------|
| 단체 분류 | ASSOCIATION / PUBLIC / PRIVATE (3개 org_type) | **단일 Entity + 자가참조 DAG** (org_type 폐지) |
| 영역 구분 | 포착 시티 = OPEN, 클럽 = CLOSED | 포착 TV / 시티 / 클럽 **3개 영역** (display_area 필드) |
| 대회 개설 | 협회(ASSOCIATION)만 가능 | **모든 단체의 ADMIN**이 개설 가능 |
| 대회 공개 | 전체 공개만 | **전체 공개 + 비공개(초대제)** 지원 |

| 가입 정책 | OPEN/CLOSED (고정) | **OPEN / APPROVAL / INVITE_ONLY** (설정 변경 가능) |

| 커뮤니티 | 미지원 | **소식/구인/모집** 지역 기반 게시판 |
| 단체 설정 | 생성 시 고정 | **매니저 권한의 "단체 설정" 화면**에서 실시간 변경 |
